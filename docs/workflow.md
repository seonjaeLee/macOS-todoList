# 개발/공개 레포 운영 규칙

## 목적

- 개발 레포와 공개 레포의 역할을 분리해 배포 실수를 줄입니다.

## 레포 역할

- 개발 레포: `macOS-todoList`
  - 기능 개발, 내부 기록, 유지보수 작업
- 공개 레포: `macOS-todoList-myfunfun`
  - 일반 사용자용 설치 가이드, 릴리스, 배포 파일 안내

## 작업 원칙

- 개발 작업은 개발 레포 루트에서만 커밋/푸시합니다.
- 공개 문서/릴리스 작업은 `public-user-repo`에서만 커밋/푸시합니다.
- 공개 레포에는 사용자에게 필요한 파일만 반영합니다.

## 커밋 전 체크

```bash
pwd
git remote -v
git status
```

위 3개가 의도한 레포를 가리킬 때만 `git add/commit/push`를 진행합니다.
